# Tech Stack

## Overview

Plateforme FX Pricing full-stack : backend Python/FastAPI pour les calculs financiers et le streaming temps réel, frontend React/TypeScript pour le dashboard et les tickets FX interactifs.

## Core Technologies

### Language

**Primary**: Python (backend) / TypeScript (frontend)
**Version**: Python 3.12+ / TypeScript 5.4+

### Framework

**Backend Framework**: FastAPI
**Version**: 0.111+
**Rationale**: Async natif (WebSocket + REST), génération OpenAPI automatique, Pydantic v2 intégré, performance élevée pour le calcul stateless.

**Frontend Framework**: React 18
**Rationale**: Reactive UI pour le streaming market data, écosystème riche pour les dashboards financiers.

### Runtime

**Backend Runtime**: CPython 3.12
**Frontend Runtime**: Node.js 20+ (build only)

## Data Layer

### Database

**Type**: Relational
**Database**: PostgreSQL 16
**Usage**: Historique des prix, calendriers de marché, conventions FX, sessions utilisateurs

### ORM / Data Access

**Tool**: SQLAlchemy 2.0 (async) + Alembic
**Rationale**: Async compatible FastAPI, migrations versionnées, support complet PostgreSQL

### Caching

**Solution**: Redis 7 — cache market data avec TTL staleness, pub/sub pour le streaming WebSocket

## API Layer

### API Style

**Style**: REST + WebSocket
- REST : pricing on-demand, CRUD conventions/calendriers, auth
- WebSocket : streaming temps réel des prix, subscriptions par paire/produit
- Server-Sent Events : fallback pour clients ne supportant pas WS

### Documentation

**Tool**: OpenAPI 3.1 (auto-généré par FastAPI) + Swagger UI

## Frontend

### UI Framework

**Framework**: React 18 + TypeScript
**Version**: React 18.3 / TypeScript 5.4

### Styling

**Approach**: Tailwind CSS 3 — utility-first, adapté aux dashboards financiers

### State Management

**Solution**: Zustand — léger, adapté au streaming market data et aux mises à jour haute fréquence

### Charts

**Library**: TradingView Lightweight Charts (graphiques FX OHLC) + Recharts (Greeks, courbes de taux)

### Real-time

**Solution**: WebSocket natif (browser API) — zero polling, subscriptions par paire FX

## Infrastructure

### Hosting

**Platform**: Docker + Docker Compose (dev) — cloud-agnostic en production

### Containerization

**Tool**: Docker — backend, frontend (nginx), PostgreSQL, Redis

### CI/CD

**Platform**: GitHub Actions
