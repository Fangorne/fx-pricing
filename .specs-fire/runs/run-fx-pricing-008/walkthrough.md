## Key Implementation Details

### Service Dependency Chain

```
redis  ─┐
         ├─ (service_healthy) ─→ backend
postgres ┘
```

Both infrastructure services must pass their healthchecks before the backend starts. This prevents connection errors on first boot.

### DATABASE_URL Construction

```yaml
environment:
  DATABASE_URL: postgresql+asyncpg://${POSTGRES_USER:-fx_user}:${POSTGRES_PASSWORD}@postgres:5432/${POSTGRES_DB:-fx_pricing}
```

Constructed inline from compose variables rather than duplicated in `.env`. `POSTGRES_PASSWORD` flows from `env_file: [.env]`, keeping the secret out of the compose file.

### Dev Volume Mount

```yaml
volumes:
  - ./backend:/app
```

The entire `backend/` directory is mounted into `/app`. Combined with uvicorn `--reload`, code changes take effect instantly without rebuilding.

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| `restart: unless-stopped` | On all services | Survives Docker daemon restart; doesn't auto-start on host boot |
| Healthcheck retries | 10 × 5s = 50s max | Postgres cold-start on first run needs time to initialize data directory |
| `start_period` for postgres | 10s | Gives postgres time to initialize before healthcheck failures count |
