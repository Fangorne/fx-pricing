## Key Implementation Details

### 1. Layer ordering for cache efficiency

```
COPY pyproject.toml uv.lock ./   ŌåÉ changes rarely ŌåÆ cached
RUN uv sync --frozen             ŌåÉ expensive, stays cached
COPY app/ ./app/                 ŌåÉ changes often ŌåÆ only this layer invalidates
```

Changing application code does not re-run dependency installation in CI.

### 2. `UV_SYSTEM_PYTHON=1`

Prevents uv from creating a virtualenv inside the container. Dependencies go directly into the system Python, saving ~20MB and eliminating path complexity.

### 3. Dev vs Prod divergence

| Aspect | Dev | Prod |
|--------|-----|------|
| Source code | Volume mount (live reload) | `COPY app/` |
| Server | uvicorn `--reload` | gunicorn + UvicornWorker |
| Dev deps | Installed | Excluded (`--no-dev`) |

### 4. `.dockerignore` scope

Excludes `.venv` (largest artefact, ~500MB), test files, caches, and editor directories. Build context sent to Docker daemon is minimal ŌĆö only `app/`, `pyproject.toml`, `uv.lock`.

## Decisions Made

| Decision | Choice | Rationale |

| Base image | `python:3.12-slim` | Minimal Debian, supports all required C extensions |
