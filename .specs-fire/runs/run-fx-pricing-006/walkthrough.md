## Key Implementation Details

### 1. `_conv()` factory for compact registry

A private `_conv()` helper removes field repetition. Each registry row is one line, matching the visual structure of a market conventions table — easy to audit against Bloomberg/Reuters references.

### 2. Input normalisation in `get_convention()`

Accepts three formats without requiring callers to import `CurrencyPair`:
- `CurrencyPair` → `str(pair)` → `"EUR/USD"`
- `"EUR/USD"` → used as-is (uppercased)
- `"EURUSD"` → split at position 3 → `"EUR/USD"`

### 3. Exception hierarchy

`FXDomainError(Exception)` → `UnsupportedCurrencyPairError` follows the coding standards pattern for explicit domain exceptions with HTTP mapping potential (400/422 in the API layer).

### 4. `DayCountBasis.ACT_ACT` included

Not used by any current G10 pair but defined for completeness — some bond-currency pairs (GBP govvies) and swaption markets use it. The pricing engine will reference it.

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Day count for GBP/USD | Act/365 | UK sterling market convention |
| Day count for EUR/USD | Act/360 | Euro money market convention |
| USD/JPY quotation | INDIRECT | 1 USD = X JPY; standard interbank convention |
| Registry key format | `"EUR/USD"` | Matches `CurrencyPair.__str__()` directly |
| `FX_CONVENTIONS` exposed | Public dict | Enables API enumeration endpoint without coupling |
