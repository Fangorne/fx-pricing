# Implementation Walkthrough: FX Spot and Value Date Generation

## Summary

Implemented `spot_date()` and `value_date()` — the two core date generation functions for FX settlement. Handles all G10 spot lags (T+1 for USD/CAD, T+2 for everything else), all standard tenor categories (ON/TN/SN, weeks, months, years), the EOM roll convention, and any `BusinessDayConvention`. 50 unit tests, 100% coverage.

## Structure Overview

`app/domain/date_generation.py` is a pure-functions module. `spot_date()` counts business days from trade date per the pair's spot lag. `value_date()` dispatches on tenor label: short-end tenors (ON/TN/SN) use trade or spot directly; week tenors add multiples of 7 days; month/year tenors use calendar arithmetic with optional EOM pinning. All adjustments delegate to `adjust()` from `business_day.py`.

## Key Implementation Details

### 1. Spot lag counting

Business days are counted by iterating forward, testing each calendar day. This correctly handles multi-day holiday clusters (e.g. Easter week) without off-by-one errors.

### 2. Month arithmetic with overflow clamping

`_add_months()` converts to a total-months count, recomputes year/month, then clamps day to the month's last day. This correctly handles Jan 31 + 1M → Feb 28 (non-leap) without exceptions.

### 3. EOM detection and pinning

Before adding a month/year period, `_is_last_bd_of_month(spot, cal)` checks if spot is the last BD of its month. If yes, the result uses `END_OF_MONTH` convention (last BD of target month) instead of `MODIFIED_FOLLOWING`. This implements the standard FX EOM roll rule.

### 4. Short-end tenor semantics

| Tenor | Settlement | Rationale |
|-------|-----------|-----------|
| ON | trade + 1 BD | Settles next business day |
| TN | trade + 1 BD | For T+2 pairs, identical to ON in practice |
| SN | spot + 1 BD | Settles one BD after spot |

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| `SPOT_LAGS` key format | `"USD/CAD"` string | Matches `CurrencyPair.__str__()` directly |
| USD always in combined calendar | Yes (except USD/CAD) | Settlement requires both financial centres open |
| `_next_bd` duplicated | Yes (private) | Avoids import dependency on `business_day` module internals |
| EOM detection threshold | `spot == last_bd_of_month(spot)` | Standard ISDA EOM rule trigger |
| Default convention | `MODIFIED_FOLLOWING` | Market standard for G10 FX |
