## Change Applied

### Missing uv cache configuration

The scaffold had `astral-sh/setup-uv@v3` without cache config. Added `enable-cache: true` and `cache-dependency-glob` keyed on `backend/pyproject.toml` + `backend/uv.lock`. This avoids re-downloading all dependencies on every CI run — critical for uv's value proposition.

Also added `if-no-files-found: warn` to the coverage artifact upload step so a missing `coverage.xml` doesn't fail the job (can happen if pytest exits early).

## Observations (No Action Needed)

- `working-directory: backend` is set at job level via `defaults.run` — cleaner than repeating it on every step.
- `uv sync --frozen --extra dev` is the correct invocation: `--frozen` ensures the lock file is authoritative; `--extra dev` includes test/lint tooling.
- Both jobs are independent — they share no outputs and can safely run in parallel. Adding `needs: [lint]` to test would serialize unnecessarily.
- `actions/setup-python@v5` before `setup-uv` ensures uv uses the pinned Python version rather than the runner's default.
