## Key Implementation Details

### Parallel jobs

```
push/PR to main
    ├── lint: ruff check → ruff format --check → mypy
    └── test: pytest --cov=app --cov-report=xml → upload coverage.xml
```

Neither job depends on the other — both start simultaneously. Total CI wall time is `max(lint_time, test_time)` rather than their sum.

### uv cache strategy

```yaml
uses: astral-sh/setup-uv@v3
with:
  enable-cache: true
  cache-dependency-glob: "backend/pyproject.toml\nbackend/uv.lock"
```

The cache is invalidated only when `pyproject.toml` or `uv.lock` changes. Application code changes (the common case) reuse the cached venv — `uv sync` becomes a no-op after the first run.

### working-directory at job level

```yaml
defaults:
  run:
    working-directory: backend
```

Applied once per job rather than repeating `working-directory:` on every step. All `run:` steps execute in `backend/` automatically.
