## Key Implementation Details

### 1. `Currency` as `enum.StrEnum`

Using `enum.StrEnum` (Python 3.11+) lets `Currency` values compare equal to plain strings (`Currency.USD == "USD"`) without a custom `__eq__`. This matters for JSON serialization (FastAPI will emit `"USD"` not `"<Currency.USD: 'USD'>"`) and for pandas/numpy interop later.

### 2. `CurrencyPair.from_string` — dual-format parsing

Normalises to uppercase and strips `/` before slicing at position 3. Handles `"EURUSD"`, `"EUR/USD"`, `"eur/usd"` uniformly. Raises `ValueError` on unknown currency codes (via `Currency(...)` constructor) or wrong length.

### 3. `Tenor` day mapping — conventional approximations

The `_TENOR_DAYS` dict uses market-standard calendar-day approximations for risk bucketing (ON=1, 1M=30, 3M=91, 1Y=365). These are NOT exact business-day counts — exact value date computation is the responsibility of the `fx-spot-value-dates` work item.

### 4. Immutability enforcement

Both `CurrencyPair` and `Tenor` use `@dataclass(frozen=True)`. Python raises `FrozenInstanceError` on attribute assignment attempts, making mutation impossible without a new instance.

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| `Currency` base class | `enum.StrEnum` | String compatibility without `__str__` override; ruff UP042 compliant |
| Day approximations | Conventional market values | Exact dates belong to the value date engine (later work item) |
| `12M` and `1Y` both map to 365 | Intentional | Both labels are valid market conventions for the 1-year tenor |
| `SN` (spot-next) | 3 days | T+3 from spot, used for ON/TN/SN roll conventions |
| No `__repr__` override | Default dataclass repr | Sufficient for debugging; `__str__` covers display |
